# Adoption Maturity Model (L0–L4)

## Purpose

This model helps organizations scale adoption without sacrificing human orchestration.

Maturity in this framework is not "more autonomy." It is better architecture, governance, and human–AI collaboration.

---

## Levels

| Level | Name | Characteristics | Human orchestration rule |
|---|---|---|---|
| **L0** | Awareness | Principles understood; no operational agent yet | Human defines architecture intent before tooling |
| **L1** | Foundation | Knowledge + governance layers documented | Human approves boundaries and first use cases |
| **L2** | Guided execution | First agents in production for bounded tasks | Human verifies core steps and consequential outputs |
| **L3** | Coordinated ecosystem | Multiple specialized agents with shared contract | Human orchestrates cross-agent sequencing |
| **L4** | Scaled governance | Portfolio-level measurement, audits, controlled evolution | Human remains strategic orchestrator; autonomy stays bounded |

---

## Advancement checks

Before moving up a level, confirm:

- source-of-truth quality is stable,
- governance controls are auditable,
- incident response is active,
- human strategic engagement is preserved.

If any fails, reinforce current level before scaling.
