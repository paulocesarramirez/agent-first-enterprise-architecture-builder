# Security Policy

## Supported Versions

This repository contains documentation and architectural frameworks only. No executable software is distributed.

| Version | Supported |
|---------|-----------|
| main    | ✅        |

## Reporting a Vulnerability

If you discover a security vulnerability within this repository's content (e.g., an example that inadvertently exposes sensitive patterns or encourages insecure practices), please report it responsibly.

**Do not open a public GitHub issue for security concerns.**

Instead, please email the maintainer directly or use GitHub's private vulnerability reporting feature:

1. Go to the **Security** tab of this repository.
2. Click **Report a vulnerability**.
3. Provide a clear description of the issue and its potential impact.

## Scope

This repository is a public documentation and architectural reference. It does not contain:

- API keys or credentials
- Executable code with runtime vulnerabilities
- Private or proprietary system configurations

If you find content that appears to accidentally expose private information, please report it immediately.

## Response Timeline

We aim to acknowledge reports within **5 business days** and resolve confirmed issues within **30 days**.
