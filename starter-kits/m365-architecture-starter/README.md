# Microsoft 365 Architecture Starter Kit

> A starting point for organizations building an agent-first architecture on Microsoft 365.

## What This Starter Kit Contains

This starter kit provides guidance for integrating the four-layer framework with Microsoft 365:

1. An M365 knowledge architecture guide
2. A Copilot governance setup checklist
3. An agent integration pattern for M365
4. A Teams-based operations rhythm template

## Who This Is For

Organizations that are:

- Already using Microsoft 365 (Teams, SharePoint, Outlook, Planner)
- Beginning to use or evaluate Microsoft 365 Copilot
- Ready to move beyond ad-hoc AI use toward a structured agent architecture
- Completed (or ready to complete) the docs-only foundation

## Prerequisites

Before using this starter kit, you should have:

- [ ] Organizational principles documented
- [ ] At least one knowledge domain structured and maintained
- [ ] An initial governance model defined (even if lightweight)
- [ ] A named owner for your first agent deployment

If you haven't completed these, start with the [`docs-only-starter`](../docs-only-starter/).

## How to Use This Starter Kit

1. Review the M365 knowledge architecture guide to understand how SharePoint should be structured
2. Work through the Copilot governance checklist before enabling Copilot broadly
3. Use the agent integration pattern to design your first M365-connected agent
4. Implement the operations rhythm template to establish a cadence for reviewing agent outputs
