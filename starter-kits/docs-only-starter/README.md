# Docs-Only Starter Kit

> A minimal starting point for organizations that want to adopt the Agent-First Architecture framework using documentation and structured knowledge — before adding any AI tooling.

## What This Starter Kit Contains

This starter kit provides the minimum viable foundation for an agent-first architecture:

1. A knowledge base template structure
2. A governance definition template
3. An agent contract template (ready to fill in)
4. A principles checklist

## Who This Is For

Organizations that are:

- Just beginning to think about AI agents in their enterprise
- Not yet using Microsoft 365 Copilot or other AI tools in production
- Wanting to establish the knowledge and governance foundation before deploying anything

## How to Use This Starter Kit

1. Copy this folder to your own repository or documentation system
2. Work through the templates in order:
   - Start with `1-principles-checklist.md` — confirm which principles apply to your organization
   - Complete `2-knowledge-base-template.md` — document your first knowledge domain
   - Fill in `3-governance-template.md` — define ownership and boundaries
   - Use `4-agent-contract-template.md` — design your first agent (even before deploying it)
3. Review with your team and iterate
4. When ready to add AI tooling, graduate to the `m365-architecture-starter` or `agent-design-starter`

## Files in This Starter Kit

| File | Purpose |
|------|---------|
| `1-principles-checklist.md` | Evaluate which framework principles apply to your org |
| `2-knowledge-base-template.md` | Template for your first knowledge domain |
| `3-governance-template.md` | Template for initial governance definitions |
| `4-agent-contract-template.md` | Template for designing your first agent |

## Next Step

Once you have completed the docs-only foundation, explore the [`m365-architecture-starter`](../m365-architecture-starter/) for platform integration guidance.
