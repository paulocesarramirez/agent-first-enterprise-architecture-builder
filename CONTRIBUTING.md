# Contributing to Agent-First Enterprise Architecture Builder

Thank you for your interest in contributing! This project is a public architectural reference and documentation framework. Contributions that improve clarity, correctness, and usefulness are welcome.

## Ways to Contribute

- **Fix typos or improve clarity** in existing documentation
- **Suggest new examples** that are safe for public sharing (no proprietary details)
- **Propose new agent design patterns** via issues before submitting a PR
- **Add or improve diagrams** in the `examples/diagrams/` folder
- **Raise questions** via the architecture-question issue template

## Ground Rules

1. **No proprietary content** — All contributions must be safe for public consumption. Do not include internal system names, credentials, or confidential business logic.
2. **Architecture-first thinking** — Contributions should reflect deliberate design thinking, not ad-hoc implementation notes.
3. **Respect the framework** — Proposals that conflict with the core principles in `docs/01-principles/` should include a clear rationale for the deviation.
4. **Be respectful** — This project follows the [Contributor Covenant](https://www.contributor-covenant.org/) code of conduct.

## Contribution Process

1. **Fork** the repository and create a feature branch from `main`.
2. **Make your changes** following the [naming conventions](docs/05-copilot-implementation/naming-conventions.md) and [markdown standards](docs/05-copilot-implementation/markdown-standards.md).
3. **Open a Pull Request** using the provided PR template.
4. A maintainer will review and respond within 10 business days.

## What We Won't Accept

- Code that executes against live systems
- Private company data or architectural diagrams
- Content that is not relevant to agent-first enterprise architecture

## Questions?

Open an issue using the [architecture-question template](.github/ISSUE_TEMPLATE/architecture-question.md).
