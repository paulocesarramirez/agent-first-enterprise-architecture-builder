# assets/visuals/

**The demo screenshots for this project do not live here.** They are stored alongside the demo documentation, in:

➡️ [`docs/04-demo/Screenshots/`](../../docs/04-demo/Screenshots/)

That folder holds the public-safe screenshots of the live EmprendHEC Microsoft 365 implementation (SharePoint knowledge architecture, agent folders, agent design documents, and agents running inside M365 Copilot). Each image is cataloged and explained in:

➡️ [`docs/04-demo/screenshots-and-diagrams.md`](../../docs/04-demo/screenshots-and-diagrams.md)

## Why this folder is (mostly) empty

To keep visuals next to the documentation that explains them, demo screenshots are kept in `docs/04-demo/Screenshots/` rather than duplicated here. This folder is retained only for **standalone visual assets that are not tied to the demo** — for example, illustrations or UI mockups referenced from multiple places.

If you only need demo imagery, use the `docs/04-demo/Screenshots/` folder above; there is no need to add files here.

## Conventions (for non-demo visuals only)

- Use descriptive filenames in kebab-case.
- Preferred formats: `.png`, `.svg`.
- Do not commit assets that contain proprietary or confidential content.
- For diagrams, use [`assets/diagrams/`](../diagrams/); for demo screenshots, use [`docs/04-demo/Screenshots/`](../../docs/04-demo/Screenshots/).
